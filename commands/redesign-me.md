Redesign the following into a **new arrangement approach** — a deliberate structural reshuffle, not an
in-place audit or bug fix: $ARGUMENTS

Use the **`design-me` skill's Redesign mode** and follow its process exactly:

1. **Inventory** the current page/view/project — every content block, control, and user task — and
   identify the **primary task**. Nothing in this inventory may be lost in the reshuffle.
2. **Right-size** the effort to the system's scope (see the skill's "Right-size first"). A single
   internal view gets a focused new layout; a whole project gets one reusable arrangement pattern
   applied consistently, not a snowflake per page.
3. Propose **2–3 genuinely distinct arrangement directions** (fewer for trivial scope) — different
   *organizations*, not restyles — each with its organizing principle, what-goes-where, and tradeoffs.
4. **Recommend one** and justify it for the actual users and scope (lean simple for internal tools).
5. **Produce the chosen arrangement in full**: new layout, regrouped/reordered content, refreshed
   hierarchy — with loading/empty/error states, responsive behavior, and accessibility intact, reusing
   the project's existing tokens and components.
6. Provide an **old → new map** so the change is reviewable and nothing was dropped.

If I already named a direction, build that one — but still inventory first and map old→new.
